# ValueCycleCode

サプライチェーンのCO₂排出量を、**数値を見せずに集計・証明する**インタラクティブデモ。

## デモ

**https://vegibusit.github.io/ValueCycleCodeDemo/**

## ValueCycleCodeとは

Scope3算定において、サプライヤーはCO₂排出量の開示を拒むことが多い。競合への情報漏洩リスク、改ざんへの不安、取引上の不利益 — 理由は様々だが、実数値の開示を求める限り、サプライチェーン全体の集計は不可能だった。

ValueCycleCode（VCC）は、ブロックチェーンで10年以上の実績がある3つの暗号技術を、**ブロックチェーンなしで**サプライチェーンに直接適用する。

| 技術 | 役割 | 出自 |
|------|------|------|
| **hyde**（TEE） | データの改ざん防止。IoTセンサーがTPMチップで電子署名 | Intel SGX等でブロックチェーンノード保護に使用 |
| **argo**（ZKP） | 数値を見せずに「基準以下」を数学的に証明 | Zcash（2016年〜）、Ethereum L2で毎日数百万件処理 |
| **plat**（FHE） | 暗号化したまま足し算。復号しても合計値しか分からない | Web3のプライバシー保護投票・オークションで実用化 |

## 「集める ≠ 見れる」— 第3のモデル

| 従来の中央集権 | ブロックチェーン | VCC |
|---|---|---|
| 集める＝見れる | 中央をなくす | **集める≠見れる** |
| → だから嫌がられる | → 管理者不在で運用困難 | → 集めるけど中身は見えない |

VCCは中央集権でも分散でもない。データを集約する管理者（小売）は存在するが、その管理者にも個社の中身は見えない。

## 競合比較

### EY OpsChain ESG（最も近い競合）

EYが2023年にベータ版をリリースしたブロックチェーンベースのScope3トラッキングプラットフォーム。ZKP（Nightfall）を使用。

| | EY OpsChain ESG | VCC |
|---|---|---|
| 基盤 | Ethereum / Polygon | **ブロックチェーンなし** |
| ZKP（証明） | ✅ あり | ✅ あり |
| FHE（暗号集計） | ❌ なし | ✅ あり |
| TEE（改ざん防止） | ❌ なし | ✅ あり |
| ガス代 | あり | **なし** |
| 暗号化したまま集計 | ❌ 復号が必要 | ✅ 暗号文のまま |
| 集計者が中身を見る | 見れる | **見えない** |

**差別化:** EYはZKPでプライバシーを守るが、集計には復号が必要。VCCはFHEにより暗号文のまま集計するため、集計者にすら個社データが見えない。

| 会社 | アプローチ | VCCとの違い |
|------|-----------|------------|
| 富士通 Green x Digital | 150社でScope3データ連携実証 | 暗号化なし、平文共有 |

**FHE + ZKP + TEE を組み合わせてScope3に適用しているプレイヤーは、2026年3月時点で確認されていない。**

## デモの構成

1. **サプライチェーン** — 農場→物流A→加工場→物流B→小売
2. **データ共有契約** — 項目ごとに「公開」「証明のみ」「暗号集計」を選択
3. **配送** — トラック走行アニメーション
4. **データ入力** — 各社が平文でCO₂を算出
5. **暗号化・送信** — 契約に従い暗号化して小売に送信（拒否する会社は途中で止まる）
6. **結果** — 集計値（従来: 不完全 / VCC: 全社集計成功）

「従来」と「VCCの世界」をタブで切り替えて比較できる。
